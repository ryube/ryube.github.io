directory

  • root/IPC, root/mEMR : html파일을 관리하는 폴더
  • root/style : css 파일을 관리하는 폴더
  • root/scripts : js 파일을 관리하는 폴더
  • root/images : 이미지를 관리하는 폴더

naming - html

  • html파일명은 의미있는 네이밍을 부여한다.(/history_emergency_detail_view.php, /history_group.php)
  • 의미가 중복되어 복잡한 경우, 숫자로 분류한다.(/devicemanagement/device_management_list.php, /devicemanagement/device_management_list02.php)

naming - css

  • 기본 표기방식은 '_' 구분의 소문자이며, 의미있는 영단어로 한다.
  • 네이밍시 기본 구조는 구조_형태_분류(ex: aside_banner_list3) 이다. 이중 구조와 분류는 필요도에 의해 제거될 수 있다.
  • 기본적인 형식은 layout, pattern을 따르며 스타일이 추가되는 경우 레이아웃 범위와 숫자로 분류한다.(contents -> contents_list -> contents_list_type1)
  • id의 사용은 앵커나 유일한 요소가 필요한 경우만 사용하며 class로 구조를 잡고, 가급적 태그명으로 제어한다.

naming - image

  • 이미지 네이밍은 GUI 가이드에서 제공된 이미지를 우선으로 사용한다.
  • 별도의 필요한 이미지 네이밍은 하기의 법칙을 따른다
    • 기본 구조는 형태_분류로 나타낸다
    • icon : icon_refresh
    • background image : bg_content, bg_content_bottom
    • Txet image : txt_contest_eidt01
  • 숫자-그룹.png 로 지정(item1-form.png)하며 변경될 수 있는 이름은 사용하지 않는다.(ex: bg, color 등의 구분)
  • 그룹명의 경우 개인에 따라 지정하지만, 기본적으로 icon의 경우 bullet, arrow, bar 등으로 분류하며, button의 경우 사이즈별(type1, type2)로 text의 경우 사용 위치(aside, navi, location)로 구분한다.
