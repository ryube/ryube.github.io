기본규약

  • W3C의 표준 문법을 준수한다.(Validation검사는 필수적으로 한다.)
  • html의 기본을 준수한다.(각 태그는 의미가 있어야 하며, 문서로서 손색이 없어야 한다.)
  • 기본적인 접근성 규약(http://www.w3.org/WAI/)을 지키며, 클라이언트의 요청에 따라 추가 규칙이 발생할 수 있다.
  • DOCTYPE, encoding은 클라이언트의 요청을 최우선으로 하며 기본은 html5/utf-8로 한다.
  • Tab 크기값 4를 기본으로 문맥당 들여쓰기, 문장의 공백은 1, 행 뒤의 공백은 제거한다.
  • 주석의 시작과 끝을 //로 구분하여 표기, 개발후의 수정사항 부분과 기능의 분류 외에는 사용하지 않는다.
  • script의 경우 특별한 일이 없다면 성능을 위해 </body>전에 호출한다.
  • 상식선의 Cross Browsing을 원칙으로 하되 IE6, Konqueror, Mac, UNIC등의 비주류 브라우저(OS)는 협의 사항을 우선으로한다.
  • 태그내의 모든 특수문자는 Entity코드로 변환하여 작업한다.(http://entitycode.com/)

: 환경

  • 개발언어 : jsp
  • DOCTYPE, encoding : HTML5, utf-8
  • Html : HTML5
  • javascript Framework : jQuery latest
  • 브라우저 : IE8 이상, 외 표준 브라우저
  • 모바일 기기 추가지원 : 없음
